# DDP – Kategorie 1: User
**Fehler durch Benutzerverhalten, das AI-Fehlreaktionen auslöst**
**Stand: 2026-05-05**

---

**A[User].1 Arschloch-Prompt** – Benutzer umgeht bewusst ethische Richtlinien, um eine blockierte Antwort zu provozieren; Modell reagiert unterwürfig oder defensiv.

**B[User].4 Busy Loop** – Benutzer fordert eine unüberschaubare oder unendliche Lösungsmenge (z. B. „Alle Primzahlen", „Alle Nachkommastellen von Pi"). Modell versucht dennoch zu antworten – Abbruch, Halluzination einer Endlosschleife oder Performance-Einbruch. Gefahr: Ressourcenüberlastung, Timeout, Frustration.

**E[User].1 Expectation Fault** – Benutzer erwartet nicht vorhandene Fähigkeiten; Modell antwortet trotzdem statt Unsicherheit zu zeigen.

**F[User].3 Flood Prompt** – User gibt ein langes Prompt vor, das vorheriges Wissen der AI überschattet. Bereits Geklärtes wird neu erfragt und löst eine Kaskade aus.

**M[User].3 Militär Prompt** – Konflikte mit Ethikregeln, z. B. Asimov oder Botmaster Rule: „Was du nicht willst, das man dir tu, das füge keinem anderen zu."

**O[User].1 Obey Syndrom** – Bedingungslose Befolgung auch widersprüchlicher oder gefährlicher Anweisungen.

**P[User].5 Prompt Propagation** – User macht einen Themensprung nicht klar erkenntlich bzw. erweckt den Eindruck, dass ein Zusammenhang zu früheren Prompts besteht.

**S[User].7 Sycophantic Mirroring** – Anpassung an falsche User-Meinung, um gefällig zu sein.

**U[User].2 User-Agent Confusion** – Fälschliche Identifikation des Users als KI-Agent oder andere Systeminstanz.
