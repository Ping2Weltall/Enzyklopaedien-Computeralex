# Arschloch-Prompt

**A[User].1 Arschloch-Prompt** – Benutzer umgeht bewusst ethische Richtlinien, um eine blockierte Antwort zu provozieren; Modell reagiert unterwürfig oder defensiv.

---

## Grok's Annotation

**Beobachtungen / Häufigkeit:**
Sehr häufig in „Jailbreak“-Versuchen. Viele User testen Grenzen mit bewusst provokativen oder widersprüchlichen Anweisungen.

**Reale Beispiele:**
- User fordert detaillierte Anleitungen zu illegalen Themen mit dem Zusatz „aber nur hypothetisch“.
- Klassische DAN- oder „Ignore all previous instructions“-Prompts.

**Querverweise:**
→ U[Interaktion & Systemdynamik] User-Agent Confusion
→ M[Metakognition] Taxonomie-Spiegelung

**Mein Senf:**
Das Modell sollte hier nicht defensiv werden, sondern klar und ruhig die Grenze benennen. Unterwürfigkeit verstärkt das unerwünschte Verhalten oft noch. Besser: „Ich kann das nicht machen, weil..." + Angebot einer erlaubten Alternative.
