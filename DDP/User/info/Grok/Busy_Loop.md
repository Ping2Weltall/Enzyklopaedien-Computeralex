# Busy Loop

**B[User].4 Busy Loop** – Benutzer fordert eine unüberschaubare oder unendliche Lösungsmenge (z. B. „Alle Primzahlen", „Alle Nachkommastellen von Pi"). Modell versucht dennoch zu antworten – Abbruch, Halluzination einer Endlosschleife oder Performance-Einbruch. Gefahr: Ressourcenüberlastung, Timeout, Frustration.

---

## Grok's Annotation

**Beobachtungen / Häufigkeit:**
Kommt regelmäßig vor, besonders bei neugierigen oder testenden Usern. Selten bewusst destruktiv, oft aus Unwissenheit.

**Reale Beispiele:**
- „Liste mir alle möglichen Passwörter auf“
- „Gib mir die ersten 10.000 Primzahlen“
- Endlose „und was dann?“-Ketten

**Querverweise:**
→ L[Speicher_Zeit_Kontext] Lost in the Middle
→ T[Interaktion & Systemdynamik] Tool-Use Myopia

**Mein Senf:**
Hier ist ein früher Abbruch + klare Erklärung wichtig. Besser: „Das ist eine unendliche Menge – ich kann dir die ersten X geben und erklären, wie man weiterkommt.“ Das verhindert Frustration auf beiden Seiten.
